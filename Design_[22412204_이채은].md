## 1. Introduction
### 1.1 Summary
우리는 일상생활 속에서 영상 스트리밍, 음원, 도서, 각종 멤버십 등 다양한 구독 서비스를 이용한다. 
하지만 종종 매달 여러 플랫폼에서 각각 결제되는 구독료의 총액이나 정확한 결제일을 파악하지 못해 곤란을 겪는 경우가 있다. 
일반적으로 개별 구독 서비스의 결제 내역과 자동 결제일 관리는 해당 서비스사 플랫폼 내부의 전유영역이다. 

그렇기 때문에 사용자들은 본인이 이용 중인 서비스들의 결제일이 언제인지, 이번 달 고정 지출 총합이 얼마인지를 각 앱에 일일이 접속하여 확인하지 않는 한 명확히 알지 못한다. 

하지만 정확한 구독 지출 사항은 주도적인 자산 관리를 하는 소비자에게 반드시 필요한 정보이다. 결제 시점을 인지하지 못해 해지 타이밍을 놓치거나, 불필요한 구독이 유지되어 망각의 비용을 지불한 경우가 종종 있을 것이다. 

위의 불편함을 해결하기 위해 산재한 구독 정보를 통합 캘린더로 가시화하고, 비필수 구독 해지 시 절약되는 비용을 실물 가치로 환산하여 보여주는 시스템 Sub-Wise를 고안했다. 

이를 통해서 소비자들이 직접 계산하지 않아도 인원별 공유 분담금과 연간 절약액을 알 수 있을 것이며, 다양한 구독 서비스에 가입되어 있어도 한 시스템에서 정보를 직관적으로 통합 관리할 수 있을 것이다.

이 문서는 Sub-Wise를 개발하기 위한 세 번째 단계인 Design에 대한 내용으로, 실제 구현에 대한 세부적인 사항을 명시하고, Diagram들을 통해 시스템의 전반적인 모습을 시각화, 순서화하여 보여준다.

### 1.2 Important Points of Design
&middot; Class 간의 정보 공유 및 데이터 통신을 수행하되, 사용자의 계정 정보 및 민감한 고정 지출 데이터가 외부로 노출되지 않도록 은닉성을 보장해야 한다.

## 3. Sequence diagram

![1_login](1_login.png)
[그림1 - Login Sequence Diagram]  
사용자가 아이디와 비밀번호를 입력하고 로그인 버튼을 클릭하면, 시스템은 등록된 사용자 정보와 일치하는지 확인한다. 이 과정에서 사용자의 권한을 일반 고객(User)과 관리자(Manager)로 식별하여 각각 권한에 맞는 메인 대시보드 화면을 출력한다. 만약 아이디나 비밀번호가 틀린 예외 상황이 발생하면, 시스템은 로그인 정보 오류 메시지를 띄우고 기존 입력 창을 유지하여 재입력을 유도한다.

## 4. State machine diagram
![StateMachineDaigram](StateDiagram.png)

### 1. 시스템 시작 및 계정 관리 단계 (Login & Register)
초기 진입: 시스템이 실행되면 가장 먼저 Login(로그인) 상태로 시작한다.

회원가입 분기: 사용자가 로그인 화면에서 회원가입 버튼(register button click)을 누르면 Register(회원가입) 화면으로 이동한다.

가입 결과 처리: 회원가입이 성공(register success)하면 시스템은 자동으로 다시 Login 화면으로 돌아간다. 반면, 가입에 실패(register fail)할 경우에는 화면이 전환되지 않고 Register 화면에 그대로 남아 사용자의 재입력을 유도한다.

### 2. 메인 대시보드 진입 및 프로필 관리 (Main Board & Profile)
로그인 성공: 로그인 검증이 성공(login success)하면 시스템의 중심 축인 Main Dash Board(메인 대시보드) 화면으로 전환된다.

프로필 및 작성 글 조회: 메인 화면에서 내 프로필 버튼(Myprofile button click)을 누르면 View Profile(프로필 조회) 화면으로 이동한다. 이 화면에서는 사용자의 기본 정보와 함께 해당 사용자가 그동안 작성한 게시글 목록(User profile Manage Page)이 연동되어 화면에 노출된다.

로그아웃 흐름: 사용자가 프로필 화면 등에서 로그아웃 버튼(Logout button click)을 누르면 Logout 상태로 전이되어 활성화된 세션을 완전히 파기하며, 로그아웃이 성공(Logout success)하는 즉시 초기 Login 화면으로 돌아간다.

### 3. 메인 대시보드 내 상시 기능 및 검색 (Main Search)
캘린더 상시 연동: 사용자는 메인 대시보드에서 캘린더 버튼(calander click)을 통해 view Calander(캘린더 조회) 화면으로 진입하여 월별 결제 예정일과 지출 금액을 상시로 확인할 수 있다.

대시보드 검색 기능: 메인 대시보드 내 검색창에 검색 값을 입력하고 검색 버튼(Search button click)을 누르면, 시스템은 데이터를 필터링하여 매칭되는 검색 결과를 화면에 즉시 반환한다.

### 4. 구독 관리 및 다이어트 시뮬레이션 (Subscription & Simulation)
구독 추가 및 수정: 메인 대시보드에서 구독 등록/수정 버튼을 누르면 Add/modify Subscription 화면으로 진입한다. 사용자가 서비스 이름, 금액, 날짜를 입력하면 병렬 처리(Fork)를 통해 create party / set shared Expense(공유 분담 설정) 기능과 Set Essential Tag(필수 태그 설정) 기능을 동시에 혹은 선택적으로 수행하여 저장할 수 있다.

다이어트 시뮬레이션: 메인 대시보드에서 시뮬레이션 버튼을 클릭하면 Diet Simulation page로 이동하며, 이어서 convert value(실물 가치 환산) 서브 상태로 진입하여 비필수 구독 해지 시 절약되는 금액을 기반으로 치킨, 커피 등의 실물 아이템 수량과 가격(Select Item, price)을 연산하여 시각적으로 출력합니다.

## 5. Implementation requirements
본 시스템은 Android Studio 2021.2.1 이상 버전을 사용하여 전체적인 어플리케이션 화면과 기능을 개발하고 빌드하였다. 스마트폰에서 실행할 수 있는 최소 사양은 Android 6.0(API Level 23) 이상이며, 내부 데이터 처리 및 분석 로직을 구현하기 위해 Pandas 2.0 라이브러리를 함께 활용할 예정이다.

이 앱은 안드로이드 네이티브 환경과 Pandas 2.0 라이브러리 버전에 맞춰서 설계되었다. 따라서 아이폰(iOS) 같은 안드로이드가 아닌 다른 운영체제나 환경에서 실행할 경우, 라이브러리 호환성이나 시스템 기능 호출 문제로 인해 특정 기능들이 정상적으로 작동하지 않을 수 있다.
